# 二级分类筛选功能修改说明

## 概述

将前端原有的"平铺式单级分类选择"升级为"两级级联分类选择"，实现市面上常见的电商分类筛选交互模式。同时配套修改后端，支持按父分类筛选其下所有子分类的商品。

---

## 1. 数据库背景

### 1.1 category 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `category_id` | NUMBER(11) PK | 分类ID |
| `category_name` | VARCHAR2(50) | 分类名称 |
| `parent_id` | NUMBER(11) FK → category | 父分类ID，NULL 表示一级分类 |
| `sort_order` | NUMBER(11) | 排序序号 |

通过 `parent_id` 自关联外键实现多级分类。

### 1.2 goods 表关键字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `category_id` | NUMBER(11) FK → category | 商品所属分类，**始终指向叶子分类** |

### 1.3 种子数据体系

```
一级分类                    二级分类
──────────────────────────────────────────
数码产品 (1)     ─┬─ 手机 (11)
                  ├─ 笔记本 (12)
                  └─ 平板/配件 (13)

教材教辅 (2)     ─┬─ 专业教材 (21)
                  ├─ 考试资料 (22)
                  └─ 文学小说 (23)

生活用品 (3)     ─┬─ 宿舍用品 (31)
                  └─ 运动健身 (32)

服饰鞋包 (4)     （无子分类）

其他 (5)         （无子分类）
```

---

## 2. 核心问题

- `goods.category_id` 指向叶子分类（如"手机"=11），**不指向父分类**（如"数码产品"=1）
- 用户选择一级分类"数码产品"时，期望看到手机+笔记本+平板/配件
- 原后端 `category_id = 1` 返回 0 条结果 → 需要支持多 ID 筛选

---

## 3. 后端修改

### 3.1 新增 `categoryIds` 参数

在保留原有 `categoryId`（单值精确匹配）的基础上，新增 `categoryIds`（逗号分隔的多值 IN 查询）。

**涉及文件（全链路）：**

| 层次 | 文件 | 修改 |
|------|------|------|
| Controller | `GoodsController.cs` | `[FromQuery] string? categoryIds = null` |
| Service 接口 | `IGoodsService.cs` | `string? categoryIds` 参数 |
| Service 实现 | `GoodsService.cs` | 透传 `categoryIds` |
| Repository 接口 | `IGoodsRepository.cs` | `string? categoryIds` 参数 |
| Repository 实现 | `GoodsRepository.cs` | 实现 IN 子句 |

### 3.2 Repository 核心逻辑

```csharp
if (!string.IsNullOrWhiteSpace(categoryIds))
{
    // categoryIds 格式: "11,12,13"
    var ids = categoryIds.Split(',', StringSplitOptions.RemoveEmptyEntries)
                         .Select(id => int.Parse(id.Trim()))
                         .ToList();
    if (ids.Count > 0)
    {
        var inParams = new List<string>();
        for (int i = 0; i < ids.Count; i++)
        {
            var paramName = $":CatId{i}";
            inParams.Add(paramName);
            parameters.Add(paramName, ids[i]);
        }
        where.Add($"category_id IN ({string.Join(", ", inParams)})");
    }
}
else if (categoryId.HasValue)
{
    // 向后兼容：原有单值精确匹配
    where.Add("category_id = :CategoryId");
    parameters.Add(":CategoryId", categoryId.Value);
}
```

### 3.3 设计考量

- `categoryIds` 优先于 `categoryId`：两者并存，`categoryIds` 有值时使用 IN 查询，否则回退到 `categoryId` 的等值查询
- 向后兼容：不改动原有 `categoryId` 参数，现有调用方不受影响
- 安全性：字符串先 `int.Parse` 后再拼接，防止 SQL 注入

### 4.3 商品列表页 — 筛选功能

**文件：** `frontend/src/views/goods/List.vue`

#### 变更前
```
[全部分类 ▼]   ← 单一下拉框，所有分类平铺
```

#### 变更后
```
[全部分类 ▼] [全部子分类 ▼]
    ↑              ↑
 一级分类      二级分类（选一级后启用）
```

#### 交互规则

| 用户操作 | 二级下拉框状态 | 发送参数 |
|----------|--------------|---------|
| 未选任何分类 | 禁用 | 不传 category 参数 |
| 选一级"数码产品"（有子分类） | 启用，显示"手机/笔记本/平板" | `categoryIds=11,12,13` |
| 再选二级"手机" | 正常 | `categoryIds=11` |
| 选一级"服饰鞋包"（无子分类） | 禁用，显示"无子分类" | `categoryIds=4` |
| 清除一级分类 | 禁用并清空 | 不传 category 参数 |

#### 关键代码

```typescript
// 一级分类变更
const handleParentChange = (value: any) => {
  selectedParentId.value = typeof value === 'number' ? value : undefined
  selectedChildId.value = undefined  // 清空二级
  page.value = 1
  fetchData()
}

// 构建筛选参数
if (selectedChildId.value) {
  params.categoryIds = String(selectedChildId.value)
} else if (selectedParentId.value) {
  if (selectedParentHasChildren.value) {
    const childIds = childCategories.value.map(c => c.categoryId)
    params.categoryIds = childIds.join(',')
  } else {
    params.categoryIds = String(selectedParentId.value)
  }
}
```

### 4.4 发布商品页 — 表单选择

**文件：** `frontend/src/views/goods/Publish.vue`

#### 变更前
```
[请选择商品分类 ▼]   ← 单一下拉，所有分类平铺，含一级+二级
```

#### 变更后
```
[请选择一级分类 ▼] [请选择二级分类 ▼]
    ↑                  ↑
 始终可见        仅当一级有子分类时出现
```

#### 交互规则

| 操作 | 行为 |
|------|------|
| 选一级"数码产品" | 右侧出现二级下拉框，`form.categoryId` 清空 |
| 再选二级"手机" | `form.categoryId = 11` |
| 选一级"服饰鞋包"（无子） | 直接 `form.categoryId = 4`，无二级下拉 |
| 切换一级分类 | 二级下拉框内容变化，`form.categoryId` 清空 |

#### 关键代码

```typescript
const handleParentChange = (value: any) => {
  const parentId = typeof value === 'number' ? value : undefined
  selectedParentId.value = parentId
  if (parentId && childCategories.value.length === 0) {
    form.categoryId = parentId  // 无子分类，直接赋值
  } else {
    form.categoryId = undefined  // 有子分类，等用户选二级
  }
}

const handleChildChange = (value: any) => {
  form.categoryId = typeof value === 'number' ? value : undefined
}
```

// 加载商品数据后，解析分类层级来回显两级选择器

})

const categoryPath = computed(() => {
  const cat = categories.value.find(c => c.categoryId === goods.value!.categoryId)
  if (!cat) return ''
  if (cat.parentId) {
    const parent = categories.value.find(c => c.categoryId === cat.parentId)
