## F01 注册及个人信息修改

**业务规则：** 用户名唯一，密码哈希存储（SHA256），注册时同时绑定学号。个人信息可修改昵称/手机/邮箱/头像。

| 层         | 文件                                  | 方法/端点                                                                 |
| ---------- | ------------------------------------- | ------------------------------------------------------------------------- |
| 前端       | `Register.vue`                      | 表单提交 →`register()`                                                 |

| Controller | `AuthController.cs`                 | `POST /api/auth/register`                                               |
| Controller | `AuthController.cs`                 | `POST /api/auth/avatar`（头像上传）                                     |
| Controller | `UsersController.cs`                | `PUT /api/users/:id`（个人信息修改）                                    |
| Service    | `AuthService.cs`                    | `RegisterAsync()` — 校验参数→密码哈希→调 Repository                  |
| Service    | `UserService.cs`                    | 修改个人信息                                                              |
| Repository | `UserRepository.cs`                 | `CreateUserWithStudentBindingAsync()` — INSERT app_user + student_auth |
| Repository | `UserRepository.cs`                 | `UsernameExistsAsync()` / `StudentIdExistsAsync()`                    |
| 表         | `app_user`                          | username, password, nickname, phone, email, avatar, role, status          |
| 表         | `student_auth`                      | student_id, real_name, college                                            |
| 存储过程   | `sp_register_user` ⚠️             | 已有但组员未用（用直接 INSERT 代替）                                      |

## F12 下单及取消订单

**业务规则（F15-F17）：** 下单时原子操作：校验购买资格→SELECT FOR UPDATE 锁商品→条件 UPDATE 防超卖→关闭该买家的活跃议价→INSERT 订单。取消时恢复商品状态。

| 层         | 文件                       | 方法/端点                                                                   |
| ---------- | -------------------------- | --------------------------------------------------------------------------- |

| Controller | `OrdersController.cs`    | `PUT /api/orders/:id/cancel`（取消）                                      |
| Service    | `OrderService.cs`        | `CreateAsync()` / `CancelAsync()`                                       |
| Repository | `OrderRepository.cs`     | `CreateAsync()` — **sp_place_order**（原子事务）                   |
| Repository | `OrderRepository.cs`     | `CancelAsync()` — **sp_cancel_order**                              |
| Repository | `OrderRepository.cs`     | `CanPurchaseAsync()` — `fn_can_purchase`                               |
| 函数       | `fn_can_purchase`        | 校验：approved + 非自买 + 未封禁 + 信用分≥0                                |
| 存储过程   | `sp_place_order`         | FOR UPDATE 行锁 → 条件 UPDATE ROWCOUNT=1 → 关买家议价 → INSERT → COMMIT |
| 存储过程   | `sp_cancel_order`        | 权限校验 → 状态校验 → 订单cancelled → 商品approved → 面交取消           |
| 表         | `trade_order`            | goods_id, buyer_id, seller_id, final_price, order_status                    |
