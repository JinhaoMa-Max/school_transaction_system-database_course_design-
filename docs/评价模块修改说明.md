# 评价模块修改说明

> 修改日期：2026-07-10  
> 分支：`bargain_renew`  
> 依据：评价实体设计 — 买卖双方完成交易后双向互评 + 首评后7天内可追评

---

## 一、修改背景

评价系统实现买卖双方在订单完成后互相评价的功能。核心规则：

### 基础互评
- 一笔已完成订单最多产生 **两条** 评价（买方评卖方、卖方评买方）
- 评价后自动更新被评价者的信用分（好评 +5，差评 −10，成交 +2）

### 追评功能（新增）
- 同一人对同一订单最多评价 **两次**（首评 + 追评各一次）
- 追评需在首评后 **7 天内** 提交，超时无法追评
- 追评与首评一样影响被评价者信用分
- 原 `UNIQUE(order_id, reviewer_id)` 约束已移除，改为存储过程层面控制上限

### 3.8 数据库存储过程 (`005_procedures.sql`)

`sp_create_review` 重写为支持首评+追评：

```sql
CREATE OR REPLACE PROCEDURE sp_create_review(
    p_order_id          IN NUMBER,
    p_reviewer_id       IN NUMBER,
    p_reviewed_user_id  IN NUMBER,   -- 新增：被评价者ID校验
    p_rating            IN NUMBER,
    p_content           IN CLOB DEFAULT NULL,
    p_review_id         OUT NUMBER
)
```

核心逻辑：

```sql
-- 查询已有评价数及首评时间
SELECT COUNT(*), MIN(created_at) INTO v_existing, v_first_review_at
FROM review WHERE order_id = p_order_id AND reviewer_id = p_reviewer_id;

IF v_existing = 0 THEN
    -- 首评：正常插入
    NULL;
ELSIF v_existing = 1 THEN
    -- 追评：需在首评后 7 天内
    IF SYSDATE - v_first_review_at > 7 THEN
        RAISE_APPLICATION_ERROR(-20023, '追评已超过7天期限，无法追评');
    END IF;
ELSE
    -- 已达上限
    RAISE_APPLICATION_ERROR(-20024, '您已达到评价次数上限（首评+追评各一次）');
END IF;
```

错误码一览：

| 错误码 | 含义 |
|--------|------|
| -20011 | 只有已完成的订单才能评价 |
| -20012 | 只有交易参与方才能评价 |
| -20022 | 被评价者与订单参与方不匹配 |
| -20023 | 追评已超过7天期限 |
| -20024 | 已达到评价次数上限 |

### 3.10 后端 Repository (`ReviewRepository.cs`)

**查询层** — 使用 Oracle `ROW_NUMBER()` 窗口函数计算 `IsFollowUp`：

```sql
SELECT ...,
    CASE WHEN ROW_NUMBER() OVER (PARTITION BY order_id, reviewer_id ORDER BY created_at) = 2
         THEN 1 ELSE 0 END AS IsFollowUp
FROM v_review_detail ...
```

**新增方法**：

### 4.1 review 表 → 前端全链路

```

─────────────────────────────────────────────────────────────────────────────────────────────

order_id (FK)            →   r.order_id           →   OrderId          →  orderId      →  orderId
reviewer_id (FK)         →   r.reviewer_id        →   ReviewerId       →  reviewerId   →  reviewerId
—                        →   rv.nickname          →   ReviewerName     →  reviewerName →  reviewerName?
reviewed_user_id (FK)    →   r.reviewed_user_id   →   ReviewedUserId   →  reviewedUserId → reviewedUserId
—                        →   rd.nickname          →   ReviewedUserName → reviewedUserName → reviewedUserName?
rating (1-5)             →   r.rating             →   Rating           →  rating       →  rating
content (CLOB)           →   r.content            →   Content          →  content      →  content?
created_at               →   r.created_at         →   CreateTime       →  createTime   →  createTime
—                        →   o.goods_id (JOIN)    →   GoodsId          →  goodsId      →  goodsId?
—                        →   g.title (JOIN)       →   GoodsTitle       →  goodsTitle   →  goodsTitle?
```

### 4.2 评价创建 INSERT 全链路（含追评）

```
前端 review/Index.vue      API POST           Service                    SP sp_create_review
─────────────────────────────────────────────────────────────────────────────────────────────
orderId                →   request.OrderId →  orderId                →  p_order_id (校验状态)
reviewedUserId         →   request.ReviewedUserId → reviewedUserId   →  p_reviewed_user_id (校验身份)
rating (1-5)           →   request.Rating  →  rating                 →  p_rating → INSERT
content (可选)         →   request.Content →  content                →  p_content → INSERT
—                         JWT 解析         →  currentUserId          →  p_reviewer_id (校验参与方)
—                         —                →  reviewCount 判断       →  v_existing 判断 (0/1/2)
—                         —                →  firstReviewDays ≤ 7?   →  SYSDATE - v_first_review_at ≤ 7?
```

### 4.3 约束与校验对应

| 校验规则 | 数据库层 | 后端层 | 前端层 |
|---------|---------|-------|-------|
| 每人每订单最多2条评价 | SP `v_existing ≥ 2` → 拒绝 | `GetReviewCountAsync ≥ 2` → 409 | `reviewCount ≥ 2` → 阻止页面 |
| 追评7天期限 | SP `SYSDATE - first > 7` → 拒绝 | `GetFirstReviewDaysAsync > 7` → 409 | 追评页面提示 |
| 评分 1-5 | `CHECK(rating BETWEEN 1 AND 5)` | Service 校验 | Arco `<a-rate :count="5">` |
| 不可自评 | `CHECK(reviewer_id <> reviewed_user_id)` | — | `reviewedUserId` = 交易对方 |
| 订单必须完成 | SP 校验 `order_status = 'completed'` | — | `status ≠ completed` → 阻止 |
| 必须是交易参与方 | SP 校验 `p_reviewer_id IN (buyer, seller)` | — | — |
