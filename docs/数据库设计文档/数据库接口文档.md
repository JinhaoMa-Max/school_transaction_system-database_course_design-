## 三、视图层（Repository 直接 SELECT）

视图为 Repository 层屏蔽了复杂的 JOIN，后端开发只需像查单表一样使用。

### 3.1 商品相关

#### v_goods_list — 商品列表（首页/搜索）

```sql
-- 后端 Repository 调用示例
SELECT goods_id, title, seller_name, category_name, price, cover_image
FROM v_goods_list
WHERE category_id = 1
ORDER BY created_at DESC;
```

| 返回字段      | 说明                     |
| ------------- | ------------------------ |
| cover_image   | 自动取该商品的第一张图片 |
| seller_name   | 自动 JOIN 了 app_user 表 |
| category_name | 自动 JOIN 了 category 表 |

#### v_goods_detail — 商品详情页

```sql
SELECT * FROM v_goods_detail WHERE goods_id = 3;
-- 返回：商品完整信息 + 卖家信用 + 是否实名 + 图片数 + 收藏数
```

### 3.2 用户相关


### 3.3 交易相关

## 四、函数层（Service 调用做校验/计算）

### 4.1 权限校验类

#### fn_can_purchase — 检查能否购买

```csharp
// Service 层调用示例（C#）
var sql = "SELECT fn_can_purchase(:userId, :goodsId) FROM dual";
var canBuy = (int)await db.ExecuteScalarAsync(sql, new { userId, goodsId });
if (canBuy == 0)
    throw new BusinessException("无法购买该商品");
```

| 参数             | 类型   | 说明              |
| ---------------- | ------ | ----------------- |
| p_user_id        | NUMBER | 买家ID            |
| p_goods_id       | NUMBER | 商品ID            |
| **返回值** | NUMBER | 1=可以 / 0=不可以 |

**校验规则：** 商品必须 approved → 不能买自己的 → 用户未被封禁 → 信用分≥0

#### fn_is_verified — 检查实名认证

```sql
SELECT fn_is_verified(userId) FROM dual;
-- 返回 1=已认证, 0=未认证
```

### 4.2 数据计算类

#### fn_avg_rating — 用户平均评分

```sql
SELECT fn_avg_rating(sellerId) FROM dual;
-- 返回 NUMBER(3,1)，无评价返回0
```

#### fn_calc_credit — 信用分重算

```sql
-- 规则：基础100 + 好评(4-5星)+5/条 - 差评(1-2星)-10/条 + 成交+2/条
SELECT fn_calc_credit(userId) FROM dual;
```

#### fn_unread_count — 未读消息数

```sql
SELECT fn_unread_count(userId) FROM dual;
-- 用于消息红点显示
```

#### fn_increment_view — 商品浏览量+1

```sql
SELECT fn_increment_view(goodsId) FROM dual;
-- 返回新的浏览量
```

### 4.3 工具类

#### fn_status_text — 状态码转中文

```sql
SELECT fn_status_text('pending', 'goods') FROM dual;  -- → '待审核'
SELECT fn_status_text('in_meet', 'order') FROM dual;  -- → '面交中'
```

#### fn_gen_confirm_code — 生成6位随机确认码


## 五、存储过程层（Service 调用做事务操作）

### 5.1 核心交易流程

#### 下单 → 面交 → 完成（完整流程）

```csharp
// Service 层调用示例（C#）
public OrderResult PlaceOrder(int goodsId, int buyerId, decimal price)
{
    var sql = @"
        BEGIN
            sp_place_order(
                p_goods_id => :goodsId,
                p_buyer_id => :buyerId,
                p_price    => :price,
                p_order_id => :orderId
            );
        END;";
  
    var param = new OracleParameter("orderId", OracleDbType.Int32, 
        ParameterDirection.Output);
    // ... execute ...
    return new OrderResult { OrderId = (int)param.Value };
}
```

**流程说明：** `sp_place_order` 内部自动完成：

1. 调用 `fn_can_purchase` 校验购买资格
2. 插入 trade_order 记录
3. 锁定商品（goods_status → 'locked'）
4. 关闭该商品所有活跃议价

#### 取消订单

```csharp
// 只能买卖双方取消，只能取消 pending_meet 状态
CALL sp_cancel_order(p_order_id => 1, p_user_id => 3);
// 内部自动：取消订单 + 恢复商品 approved + 取消关联面交
```

#### 确认面交完成

```csharp
CALL sp_complete_meet(
    p_order_id     => 2,
    p_confirm_code => '754959'  // 面交时双方核对确认码
);
// 内部自动：订单 completed + 商品 sold + 面交完成
```

### 5.2 评价流程

```csharp
CALL sp_create_review(
    p_order_id    => 2,
    p_reviewer_id => 3,   // 评价者
    p_rating      => 5,   // 1-5星
    p_content     => '学长人很好，手机完美！',
    p_review_id   => :outReviewId
);
// 内部自动：插入评价 + 更新被评价者信用分
```

### 5.3 议价流程

```csharp
// 买家发起议价
CALL sp_create_bargain(1, 4, 25.00, :offerId);

// 卖家回复
CALL sp_respond_bargain(1, 2, 'countered', 30.00);
// 可选值: 'accepted'(接受), 'rejected'(拒绝), 'countered'(还价)
```

### 5.4 管理操作

```csharp
// 审核商品
CALL sp_audit_goods(1, 3, 'approved', '商品无违规');

// 封禁用户（同时下架所有商品）
CALL sp_manage_user_ban(1, 5, 'ban', '多次违规举报属实');

// 处理举报
CALL sp_handle_report(1, 1, 'resolved', '已下架违规商品');
```

### 5.5 全部存储过程速查

| 存储过程                          | 用途          | 内部操作                       |
| --------------------------------- | ------------- | ------------------------------ |
| `sp_register_user(...)`         | 用户注册      | 重名检查 + INSERT              |
| `sp_place_order(...)`           | 下订单        | 校验→锁商品→关议价→插入订单 |
| `sp_cancel_order(...)`          | 取消订单      | 权限校验→状态校验→恢复商品   |
| `sp_complete_meet(...)`         | 完成面交      | 确认码校验→完成订单→商品sold |
| `sp_update_credit(...)`         | 更新信用分    | 调 fn_calc_credit → UPDATE    |
| `sp_create_bargain(...)`        | 创建议价      | 校验不能自议+不能重复          |
| `sp_respond_bargain(...)`       | 卖家回复议价  | 权限校验+更新状态              |
| `sp_create_review(...)`         | 交易互评      | 完成校验→插评价→更信用分     |
| `sp_audit_goods(...)`           | 商品审核      | 管理员校验→更状态→记日志     |
| `sp_handle_report(...)`         | 举报处理      | 管理员校验→更状态→记日志     |
| `sp_manage_user_ban(...)`       | 封禁/解封     | 管理员校验→更状态+下架商品    |
| `sp_send_message(...)`          | 发送消息      | 会话存在校验→插入消息         |
| `sp_get_or_create_session(...)` | 获取/创建会话 | 查已有→无则新建               |
