## 八、路由守卫自动跳转规则

| 触发条件 | 跳转目标 | 说明 |
|---|---|---|
| 已登录用户访问 `/login` 或 `/register` | `/` | 拦截已登录用户进入认证页 |
| 未登录用户访问需认证页面 | `/login?redirect=<原路径>` | 登录后返回目标页 |
| 非 admin 用户访问 `/admin/*` | `/` | 角色校验失败，跳回首页 |
| 已登录但 user 对象为空（token 有效） | 先尝试 `fetchCurrentUser()`，失败则登出 → `/login` | 恢复用户状态 |
| 开发环境 `VITE_BYPASS_AUTH=true` | 跳过所有鉴权检查 | 仅限本地开发 |

## 十、存在的问题与建议

### ⚠️ 缺少入站导航的页面

以下页面在应用中**没有任何按钮或链接可以直接到达**，只能通过手动输入 URL 访问：

| 页面 | 路径 | 严重程度 | 建议 |
|---|---|---|---|
| 收藏夹 | `/favorites` | 🔴 高 | 在首页导航栏或个人中心增加入口 |
| 议价管理 | `/bargains` | 🔴 高 | 在首页导航栏或个人中心增加入口 |
| 预约列表 | `/appointments` | 🟡 中 | 页面内容为空占位，建议补全功能或在订单列表增加入口 |

### 🔵 其他改进建议

1. **首页导航栏不完整**：已登录状态下只有 4 个入口（商品广场、发布商品、我的订单、个人中心），缺少收藏夹、议价、聊天、我的商品等常用入口。
2. **`/report` 与 `/reports` 共用同一组件**：两条路由指向同一个 `views/Report.vue`，建议确认是否为预期行为。
3. **`router.back()` 使用较多**：在部分场景下（如从外部链接直接进入），`router.back()` 可能回退到应用外部，建议对关键返回操作使用明确的路径跳转。
